Title: Getting into Javascript

Date: July 12, 2015

Previous Post | Back to Index | Next Post

Delving into some Javascript this week, I realized many things. After learning Ruby, I realized: 1) a new language is faster to learn, 2) Javascript can get confusing since concepts can be similar but not completely, 3) running into several walls again and getting past them is the usual learning process in programming, and 4) the terminology differences can get confusing.

In this post, I will cover the similarities and differences in looping between Ruby and Javascript. The ever-so important looping is an essential part in programming. Although the concept of looping is shared, and although you will see similar looping styles (i.e. for loop and while loop), the syntax is completely different.

Loops I've had exposure to in Javascript are the `for` loop, `do/while` loop, and the `while` loop. First, the `for` loop:

var i;
for (i = 0; i < 4; i++) {
 console.log(i);
};

> 0
> 1
> 2
> 3

First, I defined `var i;`. The reason why i'm defining it outside of the loop, I read it was good practice to define it outside of the for loop definition for readability and scope purposes.

When defining a for loop, we see 3 arguments separated each by semicolons within parentheses. The first argument is to initialize the loop. The second argument is an expression that tests to see when to stop looping. The last argument updates the state of the loop after each iteration.

Some new syntax: parantheses surrounding arguments, squigly brackets that define the body of the loop, semicolons to separate the arguments or end a statement or body of code, `console.log` which prints to the terminal, and `i++` which is short for `i += 1`.

Now a look at Ruby's for loop:

for i in 0..3
 puts i
end

> 0
> 1
> 2
> 3

What I noticed with Javascript that it is more bulky. You can see the difference in these `for` loops. Also, although Ruby can use `for` loops, it is un-Ruby-like and should be avoided. Rather you should use another type of loop available in Ruby, like a `times` loop or an `each` loop.

Back to Javascript, we'll take a look at a while loop. The difference between a `do/while` and a `while` loop is that the `do/while` will always run the first iteration even if it doesn't fall within the conditionals set by the `while` portion of the loop. A `while` loop by itself will always first test to see if the current code falls within the conditionals set and will only go within the body of the `while` loop if met.

I'll first show the syntax for a `do/while` loop, but won't explain it in this blog. I just want to show you the syntax.

var num = [1, 2, 3];
var sum = 0;
do {
 sum += num.pop();
} while (num.length > 0);
console.log(sum);

> 6

So the example I used for the `do/while` loop probably isn't the best use-case for it. I just wanted to show the syntax used for it. Ok, now a look at a Javascript `while` loop:

var num = [1, 2, 3];
var sum = 0;
while (num.length > 0) {
 sum += num.pop();
};
console.log(sum);

> 6

In comparison, a Ruby while loop:

num = [1, 2, 3]
sum = 0
while num.size > 0 do
 sum += num.pop
end
puts sum

> 6

While loops look very similar between the 2 languages. We see it takes a conditional, a body of code, and ends the body. While loops are nice to use when unsure of when the end goal will be met. I believe it is a nifty tool that is nice to use in both languages.

Ruby has several other loops that serve specific purposes, and I believe that Ruby code is less bulky. It requires less squigly brackets, less parantheses, no semicolons, etc. But, I understand the importance of Javascript and that it is essential in front-end web development.

I feel like I progressed well in the Javascript learning process. Still so much more to learn!
