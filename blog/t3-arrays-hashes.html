Title: Getting to Know Ruby's Array and Hash Objects

Date: June 3, 2015

Previous Post | Back to Index | Next Post

Ruby is an object oriented programming language. That means that most everyone thing in Ruby is an object. Strings are objects. Numbers (whether they be Integers or Floats) are objects. Those `true` or `false` things you see are objects as well.

And just like how all the above are objects, so are Arrays and Hashes. These 2 objects are really important parts of Ruby and you'll get to see why soon.

Let's start with Arrays. So first of all, what are Arrays anyways??
Arrays are Ruby objects that can store other Ruby objects. An analogy would be something like a refrigerator. Inside the fridge, you'll find various things like drinks, fruits & veggies, maybe some eggs, butter, and yesterday's leftovers. Well each of these items in the fridge are objects themselves. All the different types of drinks are Integers. All the different types of fruits are Boolean (true or false). Etc..

some_array = ["hello", 123, true, 7.0, "another string"];

Above is an example of an Array. An Array is always surrounded by [ brackets ]. Inside the brackets, we see that there are several different types of objects: Strings, Integer, Boolean value, and Float. Each object in an Array has an index. Indexes are basically integers that show you the position of objects inside the Array. Using the same `some_array` Array above:

some_array[0]
> "hello"
some_array[2]
> true
some_array[4]
> "another string"
some_array[-1]
> "another string"
some_array[5]
> nil
some_array[-2]
> 7.0

You can pull up an object in an Array using the above. Indexes are a little weird since they start counting from 0. So calling on index 0 on `some_array` will return the first object in the array, which is "hello". The next index would be 1, and that would return the second object in the array, which is 123. If you look a few examples below, there is `some_array[-1]`. You can count backwards starting from -1, which will always return the last object in the Array. So `some_array[-1]` returns last object "another string". Using `some_array[-2]` will return the second to last object in the array, and you can continue to count backwards to return the next object from the back (i.e. -3, -4, etc..). Using an index that doesn't exist in the array will return nil, which we can see from trying out `some_array[5]`.

There is really no limit to how much an Array can hold inside. You can even have another Array or even a Hash inside.

an_array = ["time for an array", ["this other array inside", 123, true], "outside again"]
an_array[1]
> ["this other array inside", 123, true]
an_array[1][1]
> 123

We see that `an_array` has a bunch of objects inside, including another array which also has a bunch of objects inside itself. When we pull the index of the inside array, we see that it returns that entire array. This is because the array itself is an object. So the index returns the entire object. When we try `an_array[1][1]`, we get the index of the inside array, and then the index 1 of that inside array, so it pulls a specific object within the inside array. Cool and super powerful stuff here.

So, I've been constantly talking and showing you an Array and how it can store things inside and how you can call on things from an Array. But, why is this so important? Why do we even need an Array? A big part of programming involves using objects to get other objects, or using objects and storing them somewhere so we can use them again later. Arrays are simply storage spots to store data that you plan on using again later.

Ok, so Arrays are like storage spots. What are Hashes then? Hashes are similar to Arrays, except that Hashes use what are called key/value pairs. For every key, there will be a value associated with it. Let's take a look at a Hash syntax.

a_hash = {
 "some string" => 123,
 3 => true,
 :some_symbol => "cool beans"
}

Hashes are surrounded by { these } squigly brackets. The reason why I formatted my Hash like this is so that it is easier to read. I think this is a good best practice to follow, especially if you have a very large Hash.

Hashes always have the key on the left-side, their values on the right-side, and both are separated by "=>". We can also see that each key/value pair is separated by a comma. Hashes are like Arrays in that they store data and objects of any kind, and each stored data has a value assigned to them. We can pull the value of a key like so:

a_hash["some string"]
> 123
a_hash[:some_symbol]
> "cool beans"

Instead of using indexes like we did for Arrays, we use a Hash by calling the key in order to pull out the value. Hashes have a lot of real-life uses for programs. A good example would be storing information about an individual. Let's say you have a Facebook account and you filled in a form with all your information:

  • Name: Philip Yoo
  • Gender: Coffee
  • Location: CA
  • Age: Used to be 10 years old

Well, when we fill out these forms, they can be stored in a Hash. So the above information will translate to:

user_2317 {  :name => "Philip Yoo"
 :gender => "Coffee"
 :location => "CA"
 :age => "Used to be 10 years old"
}

What is so great about the above Hash translation? Well, you have all these different fields (i.e. name, gender, etc) and it will store the user information and save this data for the user. For organization purposes, the keys will all be the same, while the values will be what is unique for each user. This is a great tool which will undoubtedly be used many times when you are programming.

This is all I have for now on Ruby Arrays and Hashes. I hope this helped as much as it helped me. Thanks for reading.
