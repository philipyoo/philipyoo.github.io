What are the differences between relative, absolute, and fixed positioning in CSS?


Posted on May 31, 2015

Previous Post | Next Post

One of the topics I had a hard time understanding was positioning concepts in CSS. Seriously, why are there even 4+ different positioning types? The more I messed around with CSS and got to test different things out, the more I realized the importance of each different positioning type.

Ok, so I will start with static positioning, which is the default if you do not give a position type. If you are writing html, writing out your content, and you have all these different blocks (i.e. paragraphs, header, footer, sections, etc), every block will appear in order by the way you wrote it out on your html file. Static is simply giving elements no positions.

Relative positioning is where elements and blocks will be positioned according to other elements on the page. If you give every element and block a relative position but you don't adjust its positioning with any other css code, it will display your page exactly as how it would have displayed your page if every element or block was given static positioning. If you adjust the position of an element (i.e. `top: -50px`), you will see that the position of this element will move relative to where it would have been with a static position.

Fixed positioning will stick the element to the page and won't budge. So if you scroll up and down, the element will always be displayed wherever you put it originally. I have seen some webpages that would have their header displayed all the time no matter how far down you scroll. This would be one use-case example for this type of positioning.

Absolute positioning is sort of like a mixture of fixed and relative. An element with an absolute position will stick wherever it is placed and won't budge. So in this case, it seems similar to a fixed position. But, absolute position will stick to the previous positioned item in your html document. So, just to help visualize this, lets say you have a relative position `<div>` in your html file. You then create an absolute position `<p>` after it. The `<p>` will position itself according to the `<div>` position on the page and will stick there.

Another example of absolute positioning.. Let's say we have all these elements in our html file but they are all static positioned. We finally add a `<p>` at the end that has absolute positioning. Since none of the previous elements are "positioned" (Remember how I related Static positioning as being given no position at all), the absolute `<p>` will position and stick to the body of the page. So it will basically turn into a Fixed position on the screen.
